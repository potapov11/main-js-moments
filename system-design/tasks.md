# Банк задач для mock System Design (frontend)

Задачи упорядочены от простых к сложным. Интервьюер выдаёт **размытое** описание, без названия паттерна.

## Уровень 1 — старт (новичок)

| ID | Размытое ТЗ | На что смотреть |
|----|-------------|-----------------|
| SD-01 | «Сделайте продающий лендинг для SaaS-продукта» | структура проекта, деплой, perf (LCP), SEO, a11y |
| SD-02 | «Todo-приложение, как в Google Tasks, но проще» | CRUD, local state vs server, оптимistic UI |
| SD-03 | «Страница профиля пользователя с аватаром и настройками» | формы, валидация, загрузка файлов, API |

## Не давать на frontend SD (или останавливать)

- Выбор брокера сообщений, шардирование БД, CAP-теорема «в лоб».
- Расчёт объёма диска под реплики PostgreSQL.
- Глубокий DevOps без связи с фронтом.

Фокус: **клиент, API как потребитель, state, perf UI, security в браузере**.
