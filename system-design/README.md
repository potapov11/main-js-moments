# System Design — подготовка к собеседованию (frontend)

Mock-интервью: сбор требований, high-level design, архитектура фронта, API, perf/security.

## Как включить

1. Открой файл из этой папки **или** упомяни `@system-design` в чате.
2. Напиши: **«начинаем system design»**, **«system design»** или **«продолжаем system design»**.

Loader Cursor: `.cursor/rules/system-design-interview.mdc`  
Содержательное правило: `.cursor/cursorrules/system-design-interview.mdc`  
Общая карта секций: `.cursor/rules/interview-modes.mdc`  
Progress tracker: `system-design/progress.md`

## Формат сессии (~45–60 мин)

1. **Сбор требований** (5–10 мин) — уточняющие вопросы, FR/NFR.
2. **High-level design** (10–15 мин) — схема системы.
3. **Детализация** (15–20 мин) — страница/фича, state, routing, API.
4. **Бонус** — performance, security, a11y (если успели).
