# Подсчёт сложности алгоритмов — с нуля

!!! info "Зачем это на алгосекции"
    На собеседовании и в LeetCode важно не только «дать ответ», но и **объяснить**, сколько времени и памяти съест решение при большом входе. Эта статья учит **считать сложность по коду** — без тяжёлой математики.

!!! tip "Практика в репозитории"
    Примеры в коде: папка `Big O notation/Notation.js`.

---

## Что такое «сложность» простыми словами

Представьте массив из **n** чисел (n — длина массива, размер задачи).

- **Временная сложность** — как **растёт время** работы, если n становится больше (10 → 100 → 10 000).
- **Пространственная сложность** — как **растёт дополнительная память** (массивы, объекты, Map), не считая сам вход.

Мы описываем не секунды на вашем ноутбуке, а **тенденцию**: «если данных в 10 раз больше, работы примерно в сколько раз больше?».

---

## Буква n — что подставлять

**n** — это размер входа в худшем (или типичном) случае:

| Задача | Что считать за n |
|--------|------------------|
| Массив чисел | `nums.length` |
| Строка | `s.length` |
| Два массива одинаковой длины | обычно одна n |
| Два массива разной длины | n и m (тогда может быть O(n·m)) |
| Дерево | число узлов |
| Граф | число вершин или рёбер (смотрите условие) |

В формулировке «O(n)» читают: **«порядка n операций»** при росте данных.

---

## Запись O(...) — Big O за 2 минуты

**O(f(n))** — верхняя оценка роста: «не хуже, чем растёт f(n)», с точностью до константы.

| Запись | Как растёт при увеличении n | Интуиция |
|--------|-----------------------------|----------|
| **O(1)** | Не зависит от n | Один шаг: взять последний элемент |
| **O(log n)** | Медленно | Делим задачу пополам (бинарный поиск) |
| **O(n)** | Линейно | Один проход по данным |
| **O(n log n)** | Чуть быстрее квадрата | Хорошие сортировки |
| **O(n²)** | Квадратично | Два вложенных полных прохода |
| **O(2ⁿ)** | Взрыв | Перебор всех подмножеств (редко на easy) |

**Что отбрасываем при подсчёте:**

- Константы: `3n` → **O(n)**
- Мелкие слагаемые: `n² + 100n` → **O(n²)**
- Операции внутри цикла, если их **фиксированное** число (не зависят от n)

---

## Пошаговый алгоритм подсчёта (шпаргалка)

1. Найдите **n** (и **m**, если два разных размера).
2. Посмотрите на **циклы** и **рекурсию** — они почти всегда задают рост.
3. Спросите: **сколько раз выполнится тело самого внутреннего цикла** в худшем случае?
4. Если внутри только фиксированные действия — умножайте/складывайте итерации.
5. Учтите **дополнительные структуры** (массив, Map, Set) — это память.
6. Запишите итог: время **O(...)** , память **O(...)** .

## Один цикл — O(n)

Каждый элемент обрабатывается **один раз**:

```javascript
function sum(nums) {
  let total = 0;
  for (let i = 0; i < nums.length; i++) {
    total += nums[i]; // O(1) на шаг
  }
  return total;
}
```

- Цикл: **n** итераций.
- Внутри: **O(1)**.
- **Итого: O(n)** по времени, **O(1)** по доп. памяти (одна переменная `total`).

---

## Два вложенных цикла — почему O(n²)

Когда **внешний** цикл идёт **n** раз и **внутренний** на каждом шаге тоже **n** раз:

```javascript
function printPairs(nums) {
  for (let i = 0; i < nums.length; i++) {
    for (let j = 0; j < nums.length; j++) {
      console.log(nums[i], nums[j]); // O(1)
    }
  }
}
```

Считаем итерации внутреннего тела:

```
n + n + n + ... (n раз) = n × n = n²
```

Это **квадратичная** сложность: в формуле **n в степени 2**. Не «два прохода» (2n), а **произведение** n на n.

**Таблица n × n** (каждая ячейка — одна итерация):

```
      j=0  j=1  ...  j=n-1
i=0    ✓    ✓         ✓
i=1    ✓    ✓         ✓
...
i=n-1  ✓    ✓         ✓
```

Всего **n²** ячеек — поэтому говорят «квадратичная».

---

## Вложенные циклы — не всегда n²

Смотрите **границы** внутреннего цикла.

### Внутренний короче: j от i до конца

```javascript
for (let i = 0; i < n; i++) {
  for (let j = i; j < n; j++) {
    // ...
  }
}
```

Итераций: `(n-1) + (n-2) + ... + 1 = n(n-1)/2` — меньше n², но главный член всё равно **n²**.  
**Итого: O(n²)** (константу ½ в Big O не пишут).

### Внутренний с фиксированным числом шагов

```javascript
for (let i = 0; i < n; i++) {
  for (let j = 0; j < 10; j++) {
    // ...
  }
}
```

**n × 10** → **O(n)** (10 — константа).

### Два разных размера: n и m

```javascript
for (let i = 0; i < n; i++) {
  for (let j = 0; j < m; j++) {
    // ...
  }
}
```

**O(n · m)**. Если m ≈ n, это снова **O(n²)**.

---

## Несколько циклов подряд (не вложенных)

```javascript
for (let i = 0; i < n; i++) { /* A */ }  // n
for (let j = 0; j < n; j++) { /* B */ }  // n
```

Складываем: **n + n = 2n** → **O(n)** (константа 2 не важна).

Берётся **доминирующий** член, если сложить разное:

- `O(n) + O(n²)` → **O(n²)**
- `O(n) + O(log n)` → **O(n)**

---

## O(1) — константное время

Работа **не растёт** с n:

```javascript
function last(nums) {
  return nums[nums.length - 1];
}
```

Даже массив из миллиона элементов — **одно** обращение по индексу (в JS это O(1) для чтения).

---

## O(log n) — делим пополам

Типичный пример — **бинарный поиск** в отсортированном массиве: на каждом шаге отбрасываем половину.

```javascript
function binarySearch(nums, target) {
  let left = 0;
  let right = nums.length - 1;

  while (left <= right) {
    const mid = Math.floor((left + right) / 2);
    if (nums[mid] === target) return mid;
    if (nums[mid] < target) left = mid + 1;
    else right = mid - 1;
  }
  return -1;
}
```

Количество шагов ≈ **log₂ n** (сколько раз можно поделить n пополам). Пишут **O(log n)**.

| n | Примерно шагов бинарного поиска |
|---|--------------------------------|
| 8 | ~3 |
| 1024 | ~10 |
| 1 000 000 | ~20 |

## Встроенные методы — не забывайте их стоимость

| Операция (массив) | Время |
|-------------------|-------|
| `push`, `pop` (с конца) | O(1) |
| `shift`, `unshift` (с начала) | O(n) — сдвиг элементов |
| `includes`, `indexOf` | O(n) |
| `sort` | O(n log n) в типичной реализации |

| Операция (Map / Set в среднем) | Время |
|------------------------------|-------|
| `get`, `set`, `has` | O(1) в среднем |
| Обход всех ключей | O(размер структуры) |

Если внутри цикла по n вызываете `nums.includes(x)` — это **O(n)** на шаг → весь код может стать **O(n²)**.

---

## Память — как считать

**O(1)** — только переменные (числа, указатели):

```javascript
let left = 0;
let right = nums.length - 1;
```

**O(n)** — создали массив/строку/Map размером с вход:

```javascript
const copy = [...nums];
const seen = new Set(nums);
```

**O(n²)** — матрица n×n или массив всех пар.

Сам входной массив в условии задачи **обычно не считают** в доп. память — смотрите, что **вы** выделяете сверху.

---

## Рекурсия — кратко

Каждый вызов — как кадр на «стеке». Если глубина вызовов **до n**:

- время часто **O(n)** (если на уровне O(1) работа);
- память **O(n)** из-за стека вызовов.

Если на каждом уровне **два** рекурсивных вызова (как наивный обход без мемоизации в некоторых задачах) — время может стать **O(2ⁿ)**. На старте достаточно помнить: **глубина и ветвление** задают рост.

---

## Типичные паттерны на алгосекции

| Паттерн в коде | Время (часто) | Память (часто) |
|----------------|---------------|----------------|
| Один проход, 2 указателя | O(n) | O(1) |
| Скользящее окно | O(n) | O(1) или O(алфавит) |
| Map/Set в одном цикле | O(n) | O(n) |
| Два вложенных цикла по массиву | O(n²) | O(1) |
| Бинарный поиск | O(log n) | O(1) |
| Сортировка + проход | O(n log n) | O(1) или O(n) |

## Мини-практика: угадайте сложность

Попробуйте сами, потом сверьтесь.

**1.**

```javascript
function f(nums) {
  let max = nums[0];
  for (const x of nums) {
    if (x > max) max = x;
  }
  return max;
}
```

<details>
<summary>Ответ</summary>

**O(n)** время, **O(1)** память.

</details>

**2.**

```javascript
function g(nums) {
  let count = 0;
  for (let i = 0; i < nums.length; i++) {
    for (let j = i + 1; j < nums.length; j++) {
      if (nums[i] === nums[j]) count++;
    }
  }
  return count;
}
```

<details>
<summary>Ответ</summary>

Пар перебирается порядка **n²/2** → **O(n²)** время, **O(1)** память.

</details>

**3.**

```javascript
function h(nums, target) {
  const set = new Set(nums);
  return set.has(target);
}
```

<details>
<summary>Ответ</summary>

Создание Set: **O(n)** время, **O(n)** память. `has` потом **O(1)**.

</details>

## Что говорить на собеседовании

Короткий шаблон:

1. «Размер входа — **n** (длина массива).»
2. «Прохожу данные **один раз** / **два вложенных цикла** → **O(n)** / **O(n²)**.»
3. «Использую **Map** размером до n → доп. память **O(n)**.»
4. «В худшем случае …» (если есть разница между средним и худшим — уточните для Hash Map: «в среднем O(1)»).

Пример:

> «Один цикл по массиву длины n, внутри операции с Map за O(1) в среднем. Время O(n), память O(n) из-за Map.»

---

## Типичные ошибки

| Ошибка | Как правильно |
|--------|----------------|
| «Два цикла → всегда O(n²)» | Два цикла **подряд** → O(n). Квадрат — когда **вложенные** и внутренний ~n раз на каждый внешний шаг |
| Считать `console.log` и мелочи | Считаем **рост** от n, не каждую строку |
| Забыть `includes` / `indexOf` внутри цикла | Это ещё один проход → умножаем сложность |
| Путать вход и доп. память | Входной массив часто не входит в «space» вашего решения |
| Писать O(n² + n) | Оставляем **O(n²)** — n² доминирует |

---

## Шпаргалка на одну страницу

```
n = размер входа (обычно длина массива/строки)

Один полный цикл          → O(n)
Два цикла подряд          → O(n)
Два вложенных по n        → O(n²)
Делим пополам каждый шаг  → O(log n)
Сортировка                → O(n log n)
Только переменные         → память O(1)
Map/массив размера n      → память O(n)

Считаем худший случай, константы выкидываем.
```
