## Минимум теории: число как биты

Число в компьютере хранится в **двоичном** виде. Например, `5` = `101`, `6` = `110`.

| Операция | Запись | Смысл |
|----------|--------|-------|
| И | `a & b` | 1 там, где оба бита 1 |
| ИЛИ | `a \| b` | 1, если хотя бы один бит 1 |
| XOR (исключающее ИЛИ) | `a ^ b` | 1, если биты **разные** |
| Сдвиг влево | `a << 1` | умножить на 2 |
| Сдвиг вправо | `a >> 1` | разделить на 2 (целочисленно) |
| NOT | `~a` | инверсия всех битов |

**XOR — главный герой на собеседовании:**

```
a ^ a = 0
a ^ 0 = a
a ^ b ^ b = a    // пары «сокращаются»
```

Порядок не важен: `a ^ b ^ c` = `c ^ a ^ b`.

## Синтаксис JavaScript: битовые операции

```javascript
// Побитовые операторы
5 & 3;    // 1  — AND
5 | 3;    // 7  — OR
5 ^ 3;    // 6  — XOR
~5;       // инверсия (осторожно со знаком в JS)

// Сдвиги
1 << 3;   // 8  — умножить на 2³
8 >> 1;   // 4  — деление на 2 (со знаком)
8 >>> 1;  // 4  — беззнаковый сдвиг (для Reverse Bits)

// i-й бит (считая с 0 справа)
(n >> i) & 1;           // прочитать
n | (1 << i);           // установить в 1
n & ~(1 << i);          // сбросить в 0
n ^ (1 << i);           // переключить

// Чётность
n & 1;                  // 0 — чёт, 1 — нечёт

// Двоичный вид (для отладки, 32 бита)
(n >>> 0).toString(2).padStart(32, '0');

// XOR строк — через charCode
'a'.charCodeAt(0) ^ 'b'.charCodeAt(0);
```

!!! warning "LeetCode и отрицательные числа"
    В JS числа — 64-bit float, но битовые ops приводят к **32-bit signed int**. Для задач с большими n используй `>>> 0` для беззнакового.

---

Число — степень двойки ⟺ в двоичной записи **ровно один** бит 1.

Примеры: 1=`1`, 2=`10`, 4=`100`, 8=`1000`.

**Трюк:** `n & (n - 1)` **обнуляет младший** установленный бит.

```
  8 = 1000
  7 = 0111
  & = 0000  → 0, значит 8 — степень двойки

  6 = 110
  5 = 101
  & = 100  → не 0, значит 6 — не степень двойки
```

```javascript
function isPowerOfTwo(n) {
  return n > 0 && (n & (n - 1)) === 0;
}
```

**O(1)** время, **O(1)** память.

## Паттерн 2: Единственное «уникальное» число (Single Number)

В массиве каждое число встречается **дважды**, кроме одного. Найти его.

**XOR всех элементов** — пары сократятся, останется уникальный:

```javascript
function singleNumber(nums) {
  let x = 0;
  for (const n of nums) {
    x ^= n;
  }
  return x;
}
```

**O(n)** время, **O(1)** память — без Map.

**Обобщение:** XOR находит симметрическую разность множеств — «что есть в одном наборе, но не в другом».

## Паттерн 3: Find the Difference

Строка `t` — это `s` + один лишний символ (перестановка). Найти лишний символ.

**Вариант XOR по кодам символов:**

```javascript
function findTheDifference(s, t) {
  let x = 0;
  for (const ch of s) x ^= ch.charCodeAt(0);
  for (const ch of t) x ^= ch.charCodeAt(0);
  return String.fromCharCode(x);
}
```

**Вариант через сумму:** `sum(t) - sum(s)` — проще, но XOR элегантнее для собеседования.

## Паттерн 4: Подсчёт единичных битов (Number of 1 Bits)

Сколько битов равны 1 в числе `n` (Hamming weight).

**Метод 1 — сдвиг:**

```javascript
function hammingWeight(n) {
  let count = 0;
  while (n !== 0) {
    count += n & 1;  // младший бит
    n >>>= 1;        // беззнаковый сдвиг вправо
  }
  return count;
}
```

**Метод 2 — Brian Kernighan** (быстрее, когда единиц мало):

```javascript
function hammingWeight(n) {
  let count = 0;
  while (n !== 0) {
    n &= n - 1;  // сбрасываем младший 1
    count++;
  }
  return count;
}
```

Каждая итерация убирает один бит 1 → **O(число единиц)**, не O(32).

## Паттерн 5: Reverse Bits

Перевернуть 32 бита числа.

```javascript
function reverseBits(n) {
  let result = 0;
  for (let i = 0; i < 32; i++) {
    result <<= 1;
    result |= n & 1;
    n >>>= 1;
  }
  return result >>> 0; // без знака
}
```

**Логика:** за 32 шага забираем младший бит из `n` и приклеиваем к `result` справа.

## Полезные приёмы — шпаргалка

| Задача | Приём |
|--------|-------|
| Чётное / нечётное | `n & 1` → 0 или 1 |
| Умножить / разделить на 2 | `<< 1` / `>> 1` |
| Проверить i-й бит | `(n >> i) & 1` |
| Установить i-й бит в 1 | `n \| (1 << i)` |
| Сбросить i-й бит | `n & ~(1 << i)` |
| Переключить i-й бит | `n ^ (1 << i)` |
| Убрать младший 1 | `n & (n - 1)` |
| Изолировать младший 1 | `n & -n` |

## Когда думать про биты

| Признак | Вероятное решение |
|---------|-------------------|
| «Каждый элемент дважды, один раз» | XOR |
| «Степень двойки» | `n & (n-1)` |
| «Переключение / toggle» | XOR |
| «Подмножество, маска состояния» | битовая маска (на medium+) |
| «Посчитать биты» | сдвиг или Kernighan |

## Типичные ошибки

| Ошибка | Как правильно |
|--------|----------------|
| Забыть `n > 0` в Power of Two | 0 и отрицательные — не степени двойки |
| `>>` vs `>>>` | Для беззнакового сдвига в JS используй `>>>` |
| XOR строк напрямую | XOR кодов символов: `charCodeAt(0)` |
| Reverse Bits: забыть ровно 32 итерации | На LeetCode фиксированная ширина 32 бита |
