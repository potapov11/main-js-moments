# Каррирование и функция `carried` (функция-ожидалка)

Эта заметка — про практический паттерн: **функция возвращает функцию**, чтобы «дособрать» аргументы и только потом вызвать исходную `fn`.

Пример из задач на собеседовании: написать `curry(fn)` или `partial(fn, ...fixed)`.

---

## Ключевые идеи

- **Rest** в параметрах собирает аргументы в массив: `(...args)`.
- **Spread** при вызове разворачивает массив в список аргументов: `fn(...args)`.
- **`fn.length`** — сколько параметров у `fn` (сколько «ожидаемых» аргументов).
- Если аргументов **мало**, мы **не вычисляем результат**, а возвращаем **функцию-ожидалку** (её часто называют `carried`, `curried`, `waitForMore`).
- Функция-ожидалка держит уже переданные аргументы в **замыкании**.

---

## Мини-скелет `curry(fn)`

```javascript
function curry(fn) {
  return function curried(...args) {
    if (args.length >= fn.length) {
      return fn(...args);
    }
    return (...next) => curried(...args, ...next);
  };
}
```

Главная строка здесь:

```javascript
return (...next) => curried(...args, ...next);
```

Это и есть **функция-ожидалка**.

---

## Что такое `carried/curried` и зачем она нужна

`curried` — функция, которая:

1. принимает **порцию** аргументов (`...args`)
2. если аргументов достаточно — вызывает `fn` и возвращает результат
3. если аргументов мало — возвращает **новую** функцию (ожидалку), которая примет следующую порцию (`...next`)

Функция-ожидалка нужна, чтобы работали цепочки вида:

```javascript
curry(sum)(1)(2)(3)
curry(sum)(1, 2)(3)
```

После `curry(sum)(1)` вы **обязаны** вернуть что-то, что можно снова вызвать как `(2)`. Это «что-то» — и есть функция-ожидалка.

---

## Где «накапливаются» аргументы

В правильной реализации нет общего массива «снаружи». Аргументы хранятся так:

- `args` живёт внутри конкретного вызова `curried(...args)`
- когда аргументов мало, возвращается функция `(...next) => ...`, и она **помнит** `args` через замыкание
- при следующем вызове `(...next)` делается «склейка» и новый заход в `curried`

Склейка:

```javascript
curried(...args, ...next)
```

Эквивалентно:

```javascript
curried(...[...args, ...next])
```

---

## Пошагово: `curry(sum)(3)(4)`

```javascript
const sum = (x, z) => x + z; // sum.length === 2
const f = curry(sum);
```

1) `f(3)`

- `args = [3]`
- `args.length = 1 < 2` → аргументов мало
- возвращаем функцию-ожидалку: `(...next) => curried(3, ...next)`

2) вызываем ожидалку: `(4)`

- `next = [4]`
- вызывается `curried(...[3], ...[4])` → `curried(3, 4)`

3) `curried(3, 4)`

- `args = [3, 4]`
- `args.length = 2 >= 2` → аргументов достаточно
- возвращаем `sum(3, 4)` → `7`

## Отладочные логи (как «увидеть» функцию-ожидалку)

Когда вы только учитесь, полезно добавить 3–4 `console.log`, чтобы видеть:

- какие аргументы пришли в `carried/curried`
- почему вернули результат или почему вернули функцию-ожидалку
- какая «следующая порция» (`next`) пришла в ожидалку

Пример (см. `interestedTasks/repeat_task_2.js`):

```javascript
function carry(fn) {
  return function carried(...args) {
    console.log('[carried] аргументы args:', args);

    if (args.length >= fn.length) {
      const result = fn(...args);
      console.log('[carried] аргументов достаточно → результат:', result);
      return result;
    }

    console.log('[carried] аргументов мало → возвращаем funcWaitMore');
    return (...next) => {
      console.log('[funcWaitMore] следующая порция next:', next);
      return carried(...args, ...next);
    };
  };
}
```

Пример вывода для `carry((x, z) => x + z)(3)(4)`:

```text
[carried] аргументы args: [ 3 ]
[carried] аргументов мало → возвращаем funcWaitMore
[funcWaitMore] следующая порция next: [ 4 ]
[carried] аргументы args: [ 3, 4 ]
[carried] аргументов достаточно → результат: 7
```

Как это читать:

- Первый вызов получил `args = [3]` и вернул функцию-ожидалку (аргументов мало).
- Ожидалка получила `next = [4]` и сделала новый вызов `carried(3, 4)`.
- Второй вызов уже смог посчитать результат и вернуть число.

## Типичные ошибки (на примере самодельного `carry`)

Если вы пишете так:

```javascript
if (args.length >= fn.length) {
  fn(args);
}
```

то это почти всегда ошибка по двум причинам:

1) `fn(args)` передаёт **один** аргумент-массив вместо нескольких аргументов  
   нужно `fn(...args)`
2) надо **вернуть** результат и прекратить цепочку на этом шаге  
   обычно делают `return fn(...args)`

Правильная ветка:

```javascript
if (args.length >= fn.length) {
  return fn(...args);
}
```

## Когда это реально используется (кроме интервью)

- **Частичная подстановка аргументов (partial application)**: заранее зафиксировать настройки
- **Фабрики функций**: `makeAdder(x)`, `makeMultiplier(x)` — чистое замыкание
- **Обёртки-утилиты**: `once(fn)`, `debounce(fn)`, `memoize(fn)`
- **Композиция обработчиков**: создать обработчик клика, который «помнит» id/настройки

Пример «фабрики»:

```javascript
function makeAdder(x) {
  return (y) => x + y;
}
```

## Мини-практика (без решения)

1) Вручную разверни: `fn(...old, ...fresh)` в `fn(?, ?, ?)`

```javascript
const old = [1, 2];
const fresh = [3];
```

2) Напиши `joinArgs(oldArgs, newArgs)` одной строкой со spread:

```javascript
joinArgs([1], [2]);       // [1, 2]
joinArgs([1, 2], [3, 4]); // [1, 2, 3, 4]
```

3) Предскажи вывод:

```javascript
const sum3 = (a, b, c) => a + b + c;
curry(sum3)(1, 2)(3);
curry(sum3)(1)(2, 3);
curry(sum3)(1, 2, 3, 99);
```
