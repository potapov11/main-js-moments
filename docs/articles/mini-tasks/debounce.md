# `debounce(fn, delay)` — теория и мини‑задача

`debounce` делает так, чтобы функция вызывалась **только после паузы**: если вызовы продолжаются — таймер постоянно сбрасывается.

Это классическая practical‑задача на **замыкание**, **таймеры** и **HOF** (функции высшего порядка).

---

## Интуиция

Пример: пользователь печатает в поле поиска.

- Без debounce — запрос уходит на **каждый** символ.
- С debounce — запрос уходит **один раз**, когда пользователь перестал печатать на `delay` мс.

---

## Что должен уметь `debounce`

Минимум:

- возвращать функцию‑обёртку
- при каждом вызове обёртки **очищать** прошлый `setTimeout`
- ставить новый `setTimeout` и по таймеру вызывать `fn`

Расширение (часто на собесе):

- `cancel()` — отменить запланированный вызов
- `flush()` — немедленно выполнить, если вызов запланирован

---

## Базовая реализация

```javascript
function debounce(fn, delay) {
  let timerId = null;

  return function (...args) {
    if (timerId !== null) clearTimeout(timerId);

    timerId = setTimeout(() => {
      timerId = null;
      fn(...args);
    }, delay);
  };
}
```

### Важные детали

- `timerId` живёт в **замыкании** (между вызовами обёртки).
- `...args` — **rest** (собрать аргументы).
- `fn(...args)` — **spread** (передать аргументы).

---

## Пример использования

```javascript
const onInput = debounce((value) => {
  console.log('search:', value);
}, 300);

onInput('r');
onInput('re');
onInput('rea');
// через ~300мс после последнего вызова выведется: "search: rea"
```

---

## Типичные ошибки

| Ошибка | Почему плохо | Как исправить |
|-------|-------------|--------------|
| Не делать `clearTimeout` | `fn` вызовется много раз | очищать прошлый таймер |
| Терять аргументы | вызывается не с теми данными | хранить последние `args` или использовать замыкание |
| Не учитывать `this` | у методов ломается контекст | использовать `fn.apply(this, args)` |

Если ожидаются методы/контекст, безопасный вариант:

```javascript
fn.apply(this, args)
```

---

## Мини‑задачи для практики

1) Реализуй `debounce(fn, delay)` с `cancel()` и `flush()`.

2) Реализуй `debouncePromise(fn, delay)`, где обёртка возвращает Promise:
- при новом вызове предыдущий промис либо отменяется (custom reject), либо все ждут один общий — проговори решение.

---

## Шпаргалка для собеседования

- `debounce` — «вызов **после паузы**; пока события идут — таймер сбрасываем».
- Внутри: `timerId` в замыкании + `clearTimeout` + `setTimeout`.
