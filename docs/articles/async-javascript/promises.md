# Основы Promise

`Promise` — объект, который представляет результат асинхронной операции. Сам результат может появиться позже: например, после ответа сервера или окончания таймера.

## Состояния

| Состояние | Значение |
|---|---|
| `pending` | Операция ещё не завершилась. |
| `fulfilled` | Операция успешно завершилась; у промиса есть значение. |
| `rejected` | Операция завершилась ошибкой; у промиса есть причина ошибки. |

`fulfilled` и `rejected` — финальные состояния: промис нельзя перевести обратно в `pending` или завершить второй раз.

## Создание промиса

```javascript
const loadUser = new Promise((resolve, reject) => {
  const succeeded = true;

  if (succeeded) {
    resolve({ id: 1, name: 'Аня' });
  } else {
    reject(new Error('Не удалось загрузить пользователя'));
  }
});
```

Функция, переданная в `new Promise`, называется executor. JavaScript вызывает её сразу. `resolve(value)` успешно завершает промис, а `reject(reason)` отклоняет его.

!!! warning "Не создавайте Promise без необходимости"
    Если API уже возвращает промис — например, `fetch()` — не нужно оборачивать его в `new Promise`. Конструктор нужен, когда вы превращаете callback API, например `setTimeout`, в промис.

## Получение результата: `.then()`, `.catch()`, `.finally()`

```javascript
fetch('/api/user/1')
  .then((response) => {
    if (!response.ok) throw new Error(`HTTP ${response.status}`);
    return response.json();
  })
  .then((user) => console.log(user.name))
  .catch((error) => console.error('Ошибка загрузки:', error.message))
  .finally(() => hideLoader());
```

- `.then(onFulfilled)` получает успешное значение.
- `.catch(onRejected)` получает ошибку из исходного промиса или из любого предыдущего `.then()`.
- `.finally(callback)` запускается в любом случае и обычно подходит для уборки: скрыть loader, вернуть кнопку в активное состояние. Он не получает результат как аргумент.

## Цепочка промисов

Каждый вызов `.then()` возвращает **новый промис**. То, что callback вернул, станет результатом следующего звена:

```javascript
Promise.resolve(2)
  .then((number) => number * 3) // новый промис со значением 6
  .then((number) => `Итог: ${number}`)
  .then(console.log); // Итог: 6
```

Если из `.then()` вернуть другой промис, следующая часть цепочки дождётся его:

```javascript
getUser()
  .then((user) => getPosts(user.id))
  .then((posts) => console.log(posts));
```

## Передача `resolve` в `.then()`

```javascript
promise2.then(resolve);
```

Это сокращённая запись:

```javascript
promise2.then((value) => {
  resolve(value);
});
```

Когда `promise2` выполнится, его значение станет значением внешнего промиса. Такой приём используется в fallback-паттерне из статьи [«Асинхронные паттерны»](async-patterns.md).

## Типичные ошибки

| Ошибка | Почему плохо | Как исправить |
|---|---|---|
| Вызвать `resolve` и затем `reject` | Учитывается только первый вызов; второй игнорируется. | Выбирайте одно финальное состояние. |
| Забыть `return` внутри `.then()` | Следующий `.then()` получит `undefined`, а не результат. | Возвращайте значение или промис. |
| Не обработать rejection | Возникает необработанная ошибка промиса. | Завершайте цепочку `.catch()` или используйте `try/catch` с `await`. |
| Оборачивать `fetch()` в `new Promise` | Это усложняет код без пользы. | Возвращайте промис `fetch()` напрямую. |

## Шпаргалка для собеседования

> Promise — это контейнер для будущего результата. Он бывает `pending`, `fulfilled` или `rejected`; после завершения состояние и значение неизменяемы. Методы `.then()` строят новую цепочку промисов, а `.catch()` ловит ошибки из всей цепочки выше.
