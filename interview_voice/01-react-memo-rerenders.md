## Вопрос

В React-приложении есть родительский компонент `Parent` и дочерний `Child`. `Parent` хранит в `useState` объект `user` и передаёт в `Child` проп `onSave`, который **каждый рендер** создаётся как новая стрелочная функция:

```tsx
function Parent() {
  const [user, setUser] = useState({ name: 'Ann', age: 30 });

  const onSave = () => {
    setUser({ ...user, age: user.age + 1 });
  };

  return <Child user={user} onSave={onSave} />;
}
```

`Child` обёрнут в `React.memo`.

**Сколько раз перерисуется `Child` при одном клике по кнопке внутри `Child`, который вызывает `onSave`? Почему именно столько?**

Ответ: число ререндеров + цепочка (Parent → Child, memo, ссылки на пропсы).

---

## Ответ кандидата (частичный)

- `Child` перерисуется **один раз**.

*Засчитана только цифра; механизм не раскрыт.*

## Критерий «зачёт»

Чёткая цепочка: клик → `setState` в `Parent` → ререндер `Parent` → новые ссылки на `user` и `onSave` → shallow compare в `memo` → ререндер `Child`.
