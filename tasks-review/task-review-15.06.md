## Задача 1 — ✅ Сделана

**167. Two Sum II — Input Array Is Sorted**

### Решение
- Два указателя `start` / `end` с краёв отсортированного массива.
- `sum < target` → `start++`, `sum > target` → `end--`, `sum === target` → вернуть `[start + 1, end + 1]` (индексация с 1).

### Оценка
| Критерий        | Результат |
|-----------------|-----------|
| Корректность    | ✅         |
| O(1) память     | ✅         |
| Время O(n)      | ✅         |

**Уровень:** уверенное решение — база для 3Sum.

## Задача 2 — ❌ Не решена самостоятельно

**15. 3Sum**

### Что было
- Условие разобрано, примеры понятны.
- Сортировка массива сделана: `[-4, -1, -1, 0, 1, 2]`.
- Алгоритм самостоятельно не подобран — функция возвращает пустой массив `[]`.
- Путаница в формулировке «избежать дубликатов»: казалось, что `[-1, -1, 2]` недопустим из‑за двух `-1` внутри триплета (на самом деле запрещены только повторяющиеся **триплеты** в ответе).
- После разбора идея ясна: фиксировать первое число + two pointers на хвосте (как Two Sum II) + пропуск дубликатов на `i`, `left`, `right`.

### Алгоритм (для повтора)
1. Отсортировать `nums`.
2. Цикл `for (i = 0; i < n - 2; i++)`.
3. Пропуск дубликата: `if (i > 0 && nums[i] === nums[i-1]) continue`.
4. Оптимизация: `if (nums[i] > 0) break`.
5. `left = i + 1`, `right = n - 1`, `target = -nums[i]`.
6. Пока `left < right`: сумма `< target` → `left++`, `> target` → `right--`, `=== target` → `push`, skip дубликатов на `left`/`right`, сдвинуть оба.
7. Вернуть `result`.

### Оценка
| Критерий              | Результат |
|-----------------------|-----------|
| Понимание условия     | ⚠️ (дубликаты) |
| Сортировка            | ✅        |
| Самостоятельный код   | ❌        |
| Алгоритм O(n²)        | ❌ (не дошёл) |

**Статус:** зафиксирована как **нерешённая** — написать решение самому без подглядывания.

## Задача 3 — ⚠️ Частично

**443. String Compression**

### Что было
- Понял ограничение **O(1)**: нельзя новый массив, объект-мапа с ростом по входу — только переменные.
- Начат паттерн **read / write**, но `read` не двигается, `write` не используется, лишний цикл `i`.
- В `else` при смене символа: `count = 0; count++` — предыдущая группа не записывается в массив.

### Алгоритм (read / write)
1. `read = 0`, `write = 0`.
2. Пока `read < chars.length`:
3. **Символ группы:** `char = chars[read]`.
4. **Длина группы:** сдвигать `read`, пока `read < length` и `chars[read] === char`; считать `count` (или `read - start`).
5. **Записать символ:** `chars[write++] = char`.
6. **Если `count > 1`:** записать цифры длины по одной в `chars[write++]`  
   (например `12` → `'1'`, `'2'`; удобно через `String(count)` и цикл по символам строки).
7. Вернуть `write` — новая длина сжатого массива.

**Пример** `["a","a","b","b","c","c","c"]`:
- группа `a`, count=2 → пишем `a`, `2`
- группа `b`, count=2 → пишем `b`, `2`
- группа `c`, count=3 → пишем `c`, `3`
- ответ: длина `6`, `["a","2","b","2","c","3", ...]`

**Сложность:** время **O(n)**, память **O(1)**.

### Оценка
| Критерий              | Результат |
|-----------------------|-----------|
| O(1) память (теория)  | ✅        |
| Паттерн read / write  | ⚠️        |
| Запись группы + count | ❌        |
| Рабочий код           | ❌        |

**Статус:** алгоритм разобран, **закрыть самому** — переписать без подглядывания.
