# Связные списки (Linked Lists) — теория для собеседований

## 📚 Что такое связный список?

Связный список — это линейная структура данных, состоящая из узлов (nodes), где каждый узел содержит:

- **Данные** (value)
- **Ссылку на следующий узел** (next)

В отличие от массива, элементы в связном списке не хранятся в непрерывной области памяти.

```javascript
// Структура узла
class ListNode {
    constructor(val, next = null) {
        this.val = val;
        this.next = next;
    }
}
```

---

## 🔄 Типы связных списков

### 1. Односвязный список (Singly Linked List)

Каждый узел ссылается только на следующий

```
[1 | next] → [2 | next] → [3 | next] → null
```

### 2. Двусвязный список (Doubly Linked List)

Каждый узел ссылается на следующий и предыдущий

```
null ← [prev | 1 | next] ↔ [prev | 2 | next] ↔ [prev | 3 | next] → null
```

### 3. Циклический список (Circular Linked List)

Последний узел ссылается на первый

```
[1 | next] → [2 | next] → [3 | next] ↑
 └──────────────────────────────────┘
```

---

## ⚡ Сравнение с массивами

| Операция | Массив | Связный список |
|----------|--------|----------------|
| Доступ по индексу | O(1) | O(n) |
| Вставка в начало | O(n) | O(1) |
| Вставка в конец | O(1)* | O(1) или O(n)** |
| Вставка в середину | O(n) | O(n) + поиск |
| Удаление из начала | O(n) | O(1) |
| Удаление из конца | O(1)* | O(1) или O(n)** |
| Поиск элемента | O(n) | O(n) |
| Память | Непрерывная | Фрагментированная |

* — если есть свободное место в конце  
** — O(1) если есть ссылка на хвост (tail), иначе O(n)

---

## 🎯 Когда использовать связные списки?

**Используем, когда:**

- Нужно часто вставлять/удалять в начале списка
- Неизвестен точный размер данных
- Нужна структура с динамическим размером
- Реализация стеков и очередей

**НЕ используем, когда:**

- Нужен частый доступ по индексу
- Важна кеш-локальность (массивы быстрее из-за непрерывности)
- Данные нужно часто сортировать

## 🛠 Основные операции с реализациями

### Обход списка (Traversal)

```javascript
function traverse(head) {
    let current = head;
    while (current !== null) {
        console.log(current.val);
        current = current.next;
    }
}
```

### Поиск элемента (Search)

```javascript
function search(head, target) {
    let current = head;
    while (current !== null) {
        if (current.val === target) return current;
        current = current.next;
    }
    return null;
}
```

### Вставка в начало (Insert at beginning)

```javascript
function insertAtBeginning(head, val) {
    const newNode = new ListNode(val);
    newNode.next = head;
    return newNode; // новый head
}
```

### Вставка в конец (Insert at end)

```javascript
function insertAtEnd(head, val) {
    const newNode = new ListNode(val);
    
    if (head === null) return newNode;
    
    let current = head;
    while (current.next !== null) {
        current = current.next;
    }
    current.next = newNode;
    return head;
}
```

### Вставка в середину (Insert at position)

```javascript
function insertAtPosition(head, val, position) {
    const newNode = new ListNode(val);
    
    if (position === 0) {
        newNode.next = head;
        return newNode;
    }
    
    let current = head;
    let index = 0;
    
    while (current !== null && index < position - 1) {
        current = current.next;
        index++;
    }
    
    if (current === null) return head; // позиция вне диапазона
    
    newNode.next = current.next;
    current.next = newNode;
    return head;
}
```

### Удаление из начала (Delete from beginning)

```javascript
function deleteFromBeginning(head) {
    if (head === null) return null;
    return head.next; // новый head
}
```

### Удаление из конца (Delete from end)

```javascript
function deleteFromEnd(head) {
    if (head === null || head.next === null) return null;
    
    let current = head;
    while (current.next.next !== null) {
        current = current.next;
    }
    current.next = null;
    return head;
}
```

### Удаление по значению (Delete by value)

```javascript
function deleteByValue(head, target) {
    if (head === null) return null;
    
    // Если удаляем голову
    if (head.val === target) return head.next;
    
    let current = head;
    while (current.next !== null) {
        if (current.next.val === target) {
            current.next = current.next.next;
            return head;
        }
        current = current.next;
    }
    
    return head; // значение не найдено
}
```

## 🧠 Паттерны и техники

### Два указателя (Two Pointers)

- **Медленный и быстрый** (slow/fast) — для обнаружения циклов и поиска середины
- **Разведчик и основной** (runner/current) — для задач, где один указатель отстаёт

### Фиктивная голова (Dummy Node)

```javascript
const dummy = new ListNode(0);
dummy.next = head;
// работаем с dummy.next, это упрощает обработку краевых случаев
// (например, когда нужно удалить или вставить элемент в начало)
```

### Рекурсивный подход

```javascript
// Пример рекурсивного обхода
function traverseRecursive(head) {
    if (head === null) return;
    console.log(head.val);
    traverseRecursive(head.next);
}
```

Рекурсия часто используется для:

- Разворота списка
- Сравнения списков
- Задач, где нужно идти от конца к началу

## ⚠️ Типичные ошибки на собеседованиях

| Ошибка | Решение |
|--------|---------|
| Потеря ссылки на следующий узел при вставке/удалении | Всегда сохранять следующую ссылку перед изменением |
| Null pointer exception — обращение к next у null | Всегда проверять на null перед next |
| Бесконечный цикл при обходе | Убедиться, что указатель двигается и есть условие выхода |
| Неучёт краевых случаев | Всегда проверять: пустой список, один элемент, два элемента |
| Неправильная обработка головы при вставке/удалении | Не забывать возвращать новый head, если он изменился |

## 🔑 Ключевые фразы для собеседования

- «Использую два указателя: медленный и быстрый»
- «Создам фиктивную голову (dummy node), чтобы избежать краевых случаев»
- «Здесь нужно быть внимательным, чтобы не потерять ссылку на следующий узел»
- «Сначала обработаю краевые случаи: пустой список и список из одного элемента»
- «В двусвязном списке нужно обновить обе ссылки: next и prev»
- «Поскольку у нас нет прямого доступа по индексу, приходится проходить список последовательно»

## 💡 Советы по практике

1. Рисуй списки на бумаге — это помогает визуализировать, куда указывают ссылки
2. Всегда проверяй краевые случаи: null, один элемент, два элемента
3. Проходи по шагам с маленькими примерами, записывая состояния указателей
4. Начинай с brute force, потом оптимизируй
5. На собеседовании проговаривай вслух, что делаешь с указателями — это показывает ход мыслей
6. Помни про память: в JS сборщик мусора, но в других языках нужно думать об удалении узлов

## 📊 Сравнение с другими структурами

| Характеристика | Массив | Связный список | Дерево | Хеш-таблица |
|----------------|--------|----------------|--------|-------------|
| Упорядоченность | По индексу | По связям | Иерархическая | Нет |
| Произвольный доступ | O(1) | O(n) | O(log n)* | O(1)** |
| Вставка/удаление | O(n) | O(1)*** | O(log n) | O(1) |
| Доп. память | Нет | На указатели | На указатели | На хеш и коллизии |

* — в сбалансированном дереве  
** — в среднем случае  
*** — если известна позиция

## 🎯 Где применяются в реальной жизни

- **Браузер:** история навигации (back/forward) — двусвязный список
- **Плеер:** плейлист с переходом к следующему/предыдущему треку
- **Фоторедактор:** история действий (undo/redo)
- **Очереди задач:** FIFO на основе односвязного списка
- **Кэширование:** LRU-кэш (Least Recently Used) на основе двусвязного списка
